## Empathy Map Canvas

An empathy map is a simple, easy-to-digest visual that captures knowledge about a user's behaviours and attitudes. It is a useful tool to helps teams better understand their users.

Creating an effective solution requires understanding the true problem and the person who is experiencing it. The exercise of creating the map helps participants consider things from the user's perspective along with his or her goals and challenges.

## Empathy Map for Quote Chatbot Users

### What Does the User THINK AND FEEL?

- **What really counts:** Getting the right quote at the right time
- **Major preoccupations:** Finding motivation for daily challenges
- **Worries & Aspirations:** Wants to feel inspired, fears lack of motivation

### What Does the User HEAR?

- **What friends say:** "You should read more quotes"
- **What influencers say:** Motivational content on social media
- **What the market offers:** Various quote apps with limited features

### What Does the User SEE?

- **Environment:** People using smartphones for self-improvement
- **Friends:** Sharing inspirational quotes on social platforms
- **What the market offers:** Basic quote applications with search functionality

### What Does the User SAY AND DO?

- **Attitude in public:** Seeks positive reinforcement
- **Appearance:** Engaged with self-improvement content
- **Behavior towards others:** Shares motivational content

## Pain Points and Gain Points

### Pain Points (Frustrations, Fears, Obstacles)

| Pain Point | Description |
|------------|-------------|
| Time-consuming search | Users spend too much time looking for relevant quotes |
| Lack of personalization | Generic recommendations don't match user needs |
| No conversational access | Must navigate through menus and filters |
| Information overload | Too many quotes to choose from |
| No feedback loop | Can't indicate if a quote was helpful |

### Gain Points (Wants, Needs, Measures of Success)

| Gain Point | Description |
|------------|-------------|
| Quick access | Get quotes instantly through conversation |
| Personalized recommendations | Quotes tailored to emotional state |
| Natural interaction | Chat-based interface feels human |
| Categorized content | Easy access to specific quote types |
| Feedback mechanism | Can rate quote satisfaction |

## Steps in User Journey

### Step 1: Initial Contact

| Aspect | Description |
|--------|-------------|
| **What does the person experience?** | User opens the chatbot application |
| **Interactions** | Clicks on web interface or app icon |
| **Things** | Smartphone, web browser |
| **Places** | Home, office, anywhere with internet |
| **People** | User alone |
| **Goal** | "Help me find motivation" |

### Step 2: Request for Quote

| Aspect | Description |
|--------|-------------|
| **What does the person experience?** | User types a request like "I need motivation" |
| **Interactions** | Text input in chat interface |
| **Things** | Chatbot with NLP capability |
| **Places** | Within the application |
| **People** | User interacting with bot |
| **Goal** | "Help me get an inspiring quote" |

### Step 3: Quote Reception

| Aspect | Description |
|--------|-------------|
| **What does the person experience?** | Receives personalized quote based on intent |
| **Interactions** | Reads the quote response |
| **Things** | Quote displayed with author name |
| **Places** | Chat interface |
| **People** | Bot providing content |
| **Goal** | "Show me a quote that resonates" |

### Step 4: Feedback

| Aspect | Description |
|--------|-------------|
| **What does the person experience?** | Provides feedback on the quote |
| **Interactions** | Types "thank you" or "not helpful" |
| **Things** | Feedback mechanism in chat |
| **Places** | Within conversation |
| **People** | User expressing satisfaction/dissatisfaction |
| **Goal** | "Let the system know if it helped" |

## Positive and Negative Moments

### Positive Moments

- **Enjoyable:** Receiving a quote that perfectly matches the mood
- **Productive:** Getting inspiration quickly without searching
- **Fun:** Interactive conversation with the chatbot
- **Motivating:** Reading impactful words from famous authors
- **Delightful:** Unexpected variety in quote recommendations

### Negative Moments

- **Frustrating:** When the bot doesn't understand the intent
- **Confusing:** Too many options to choose from
- **Angering:** Receiving irrelevant quotes
- **Costly:** Time wasted on ineffective searches
- **Time-consuming:** Manual filtering through quotes

## Areas of Opportunity

### How Might We Improve Each Step?

| Step | Opportunity |
|------|-------------|
| Initial Contact | Make the interface inviting and easy to access |
| Request | Improve NLP to understand varied phrasing |
| Quote Reception | Add more quote variations per category |
| Feedback | Implement smart follow-up suggestions |

### Ideas from Research

1. **Implement Rasa NLU** for better intent recognition
2. **Add sentiment analysis** to gauge user mood
3. **Create conversation stories** for natural flow
4. **Build feedback loop** for continuous improvement
5. **Expand quote database** with diverse categories

---

## Summary

This empathy map reveals that users want a **conversational, intelligent, and personalized** quote recommendation system. The Quotes Recommendation Chatbot using Rasa NLP addresses these needs by providing:

- Natural language understanding
- Instant quote delivery
- Category-based filtering
- User feedback integration
